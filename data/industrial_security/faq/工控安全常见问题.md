# 工控安全常见问题

## 基础概念

### Q1: 什么是工控安全？与传统 IT 安全有什么区别？

**工控安全**（ICS Security）是指保护工业控制系统免受网络威胁的安全措施。

**主要区别：**

| 方面 | IT 安全 | 工控安全 |
|------|--------|---------|
| 首要目标 | 机密性（CIA 中的 C） | 可用性（CIA 中的 A） |
| 系统生命周期 | 3-5 年 | 15-20 年 |
| 补丁策略 | 及时打补丁 | 需充分验证后打补丁 |
| 停机容忍度 | 可计划停机 | 尽量不停机 |
| 协议类型 | TCP/IP 为主 | 工业协议（Modbus、S7 等） |
| 实时性要求 | 一般 | 高（毫秒级） |

### Q2: 什么是等保 2.0 工控扩展要求？

等保 2.0 针对工业控制系统增加了专门的扩展要求，主要包括：

1. **网络架构**：工控系统与企业其他系统隔离
2. **通信传输**：工业协议完整性校验
3. **入侵防范**：检测工业协议异常
4. **恶意代码防范**：移动介质安全管理
5. **安全审计**：工控操作日志记录

### Q3: 什么是 IEC 62443 标准？

IEC 62443 是工业自动化和控制系统安全的国际标准系列，由 ISA/IEC 联合制定。

**标准结构：**
- **62443-1-x**：通用概念和术语
- **62443-2-x**：安全管理政策和程序
- **62443-3-x**：系统安全要求
- **62443-4-x**：组件安全要求

**安全等级（SL）**：
| 等级 | 防护目标 |
|------|---------|
| SL1 | 防护偶发违规 |
| SL2 | 防护简单攻击 |
| SL3 | 防护复杂攻击 |
| SL4 | 防护国家级攻击 |

## 产品相关

### Q4: 态势感知系统与传统 SIEM 有什么区别？

**工控态势感知**相比传统 SIEM 的增强：

| 能力 | 传统 SIEM | 工控态势感知 |
|------|---------|-------------|
| 协议解析 | IT 协议 | IT+OT 协议 |
| 资产识别 | 服务器/PC | PLC/DCS/HMI 等 |
| 威胁检测 | 基于规则 | 规则+行为基线+AI |
| 可视化 | 网络拓扑 | IT/OT 融合视图 |

### Q5: AI-Fence 工业防火墙与普通防火墙的区别？

| 特性 | 普通防火墙 | AI-Fence |
|------|---------|----------|
| 协议识别 | 应用层协议 | 50+ 工业协议深度解析 |
| 检测方式 | 特征匹配 | AI 行为分析 |
| 延迟要求 | 毫秒级可接受 | 微秒级（<10μs） |
| 部署方式 | 路由/NAT | 透明桥接/旁路 |
| 工业特性 | 无 | 功能码过滤、值域校验 |

### Q6: 为什么需要专门的 USB 安全工作站？

**原因：**
1. **U 盘是主要感染途径**：80% 工控病毒通过 U 盘传播
2. **不能装杀毒软件**：工控主机无法安装杀毒软件
3. **物理隔离**：安全工作站独立于生产网络
4. **多引擎检测**：单一引擎检出率约 60%，多引擎可达 99.9%
5. **审计管理**：完整记录所有介质使用

## 实施相关

### Q7: 工控安全项目如何不影响生产？

**最佳实践：**

1. **充分调研**：理解业务流程和系统架构
2. **旁路部署**：先旁路监控，确认无影响后切换
3. **分批实施**：按区域/产线分批部署
4. **测试验证**：每个环节充分测试
5. **回退预案**：准备快速回退方案
6. **窗口期实施**：利用维护窗口进行切换

### Q8: 老旧系统（Windows XP/2003）如何防护？

**解决方案：**

| 方案 | 说明 | 优先级 |
|------|------|--------|
| 网络隔离 | 使用工业防火墙隔离 | 高 |
| 应用白名单 | 只允许已知程序运行 | 高 |
| 主机加固 | 关闭不必要服务 | 中 |
| 虚拟补丁 | IPS/防火墙提供防护 | 中 |
| 升级替换 | 条件允许时升级 | 长期 |

### Q9: 项目实施周期一般多长？

**典型周期：**

| 项目规模 | 周期 | 说明 |
|---------|------|------|
| 单产线 | 4-6周 | 单系统防护 |
| 单工厂 | 3-6月 | 完整防护体系 |
| 集团级 | 6-12月 | 多基地统一管理 |

**主要阶段：**
1. 需求调研（2-4周）
2. 方案设计（2-4周）
3. 设备部署（4-8周）
4. 策略配置（2-4周）
5. 试运行（2-4周）
6. 正式上线

## 服务相关

### Q10: 渗透测试会影响生产系统吗？

**安全保障措施：**

1. **测试环境**：优先使用测试/仿真环境
2. **时间窗口**：与生产计划协调
3. **非破坏性**：使用安全测试工具
4. **实时监控**：测试期间持续监控
5. **紧急回退**：准备应急措施
6. **专业团队**：有工控经验的测试人员

## 成本相关

### Q12: 工控安全投资回报如何计算？

**ROI 计算方法：**

```
年度收益 = 避免损失 + 合规达标价值 + 效率提升
避免损失 = P(事故) × 事故损失 × 防护有效率
ROI = (年度收益 - 年度成本) / 投资总额 × 100%
```

**典型收益项：**
| 收益项 | 估算方法 |
|--------|---------|
| 避免停产 | 日产值 × 预估停产天数 |
| 避免赎金 | 行业平均赎金 |
| 合规达标 | 避免罚款/准入 |
| 保险折扣 | 保费降低 |

### Q13: 安全投入占 IT 预算多少比例合适？

| 行业 | 建议比例 | 说明 |
|------|---------|------|
| 关键基础设施 | 10-15% | 电力、石化、交通 |
| 一般制造业 | 5-10% | 汽车、电子、机械 |
| 轻工业 | 3-5% | 食品、纺织 |
