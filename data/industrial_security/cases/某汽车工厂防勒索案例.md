# 某汽车工厂勒索病毒防护及应急响应案例

## 客户背景

- **客户名称**：某合资汽车制造企业
- **行业**：汽车制造
- **规模**：年产 30 万辆，员工 8000 人
- **系统范围**：MES 系统、机器人控制系统、AGV 调度系统

## 事件背景

### 安全事件

2023 年某月，客户工厂遭遇勒索病毒攻击，部分产线被迫停产。紧急启动应急响应，同时进行安全体系建设。

### 事件经过

| 时间 | 事件 |
|------|------|
| T+0 04:00 | MES 服务器告警，部分文件被加密 |
| T+0 06:00 | 发现勒索信息，确认为 LockBit 变种 |
| T+0 08:00 | 启动应急预案，隔离受影响区域 |
| T+0 12:00 | 安全团队到场，开始取证分析 |
| T+1 | 确认攻击路径，清除残留威胁 |
| T+3 | 从备份恢复关键系统 |
| T+5 | 产线恢复生产 |

### 损失评估

| 损失类型 | 金额 |
|---------|------|
| 停产损失 | ¥2,000万（5天） |
| 恢复成本 | ¥200万 |
| 应急服务 | ¥80万 |
| **总计** | **¥2,280万** |

### 攻击路径分析

```
攻击路径还原:

外部钓鱼邮件
    ↓
办公网 PC 感染
    ↓
内网横向移动（利用弱密码）
    ↓
IT/OT 边界突破（无隔离）
    ↓
MES 服务器感染
    ↓
文件加密 + 勒索
```

### 根因分析

| 问题 | 影响 |
|------|------|
| 邮件安全薄弱 | 钓鱼邮件未被拦截 |
| 弱密码 | 横向移动无阻碍 |
| IT/OT 未隔离 | 攻击蔓延至生产系统 |
| 备份不完善 | 恢复时间长 |

## 安全体系建设

### 事后整改方案

基于事件教训，建立完整的工控安全防护体系：

| 防护层 | 措施 | 产品 |
|--------|------|------|
| 边界防护 | IT/OT 严格隔离 | AI-Fence 5000 × 4 |
| 介质管控 | U盘全面管控 | USB-Guard Pro × 6 |
| 终端防护 | 应用白名单 | 终端安全软件 |
| 监控告警 | 7×24 监控 | SIEM-ICS 3000 |
| 数据备份 | 每日备份+异地容灾 | 工业备份系统 |
| 应急预案 | 完善应急流程 | 应急服务 |

### 实施效果

| 能力 | 建设前 | 建设后 |
|------|--------|--------|
| 边界防护 | 无 | AI 深度检测 |
| 威胁检测 | 被动发现 | 实时告警 |
| 恢复能力 | 5天 | 4小时 |
| 应急响应 | 无预案 | 完善流程 |

## 经验教训

### 安全建设建议

1. **边界隔离**：IT/OT 必须严格隔离
2. **纵深防御**：单一措施不足，需要多层防护
3. **数据备份**：定期备份、验证恢复、异地存储
4. **安全意识**：加强员工安全培训
5. **应急演练**：定期演练，确保预案有效
